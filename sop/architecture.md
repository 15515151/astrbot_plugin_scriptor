# 核心架构 SOP

## 1. 记忆存储原则
- **文件即记忆**：所有长期记忆必须以 Markdown 格式存储在 `profiles/` 或 `groups/` 目录下。
- **透明性**：用户应能直接阅读和编辑这些文件。
- **原子性**：每条记忆应尽可能独立，便于检索和更新。

## 2. 身份识别流程
- 物理 ID (Platform + ID) -> 逻辑 UID。
- 支持跨平台绑定，通过 `/bind` 命令实现。

## 3. 检索策略
- **Entity-First**：优先检索个人画像和核心规则。
- **混合检索**：Tantivy BM25 (关键词) + ChromaDB (向量)。
- **时间衰减**：根据 `useful_score` 应用三档衰减策略。
