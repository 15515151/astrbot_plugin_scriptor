## 🎯 三态认知模型

| 状态 | 图标 | 特点 | 使用场景 |
|------|------|------|---------|
| **日常模式** | 🟢 | 正常工作，知识可读写 | 日常对话、任务处理 |
| **学习模式** | 🟡 | 积极提取，待确认机制 | 学习新知识、业务培训 |
| **授课模式** | 🔴 | 只读锁定，权威专家 | 教学、答疑、知识输出 |

## 🟡 学习模式详解

### 核心特点
- ✅ **积极提取**: AI 主动从对话中提取知识点
- ✅ **待确认机制**: 知识不会立即写入数据库
- ✅ **防止污染**: 避免错误知识进入知识库
- ✅ **双轨写入**: 确认后同时写入知识库和知识图谱

### 典型使用流程
```
1. /开始学习              # 进入学习模式
2. 进行教学对话            # AI 提取知识点
3. /学习状态              # 查看待确认知识
4. （可选）修改知识         # 修正错误
5. /确认知识              # 写入数据库
6. /结束学习              # 退出学习模式
```

### 示例对话
```
/开始学习

HR: 我们公司的工作时间是早上 9 点到下午 6 点，午休 1 小时。
AI: 好的，已记录"公司工作时间"知识点。

HR: 请假流程需要先在 OA 系统提交，然后直属领导审批。
AI: 明白，已记录"请假流程"知识点。

HR: 还有，报销需要在每月 25 号前提交。
AI: 好的，"报销截止时间"也已记录。

/学习状态
# 📊 学习状态详情
# 当前状态：🟡 学习模式
# 待确认知识：3 条

/确认知识
# ✅ 已确认 3 条知识
# ✅ 写入知识库
# ✅ 写入知识图谱

/结束学习
# ✅ 已切换回日常模式 🟢
```

### 推荐场景
- ✅ 系统性业务培训（如公安办案流程）
- ✅ 领域知识学习（如法律、医疗）
- ✅ SOP 标准操作流程
- ✅ 组织规章制度
- ✅ 新产品知识培训

## 🔴 授课模式详解

### 核心特点
- 🔒 **知识库只读**: 不允许修改知识库
- 🎓 **专家身份**: AI 以权威专家回答
- ✅ **适合教学**: 答疑、考核、咨询
- ⚠️ **严格模式**: 可配置是否完全禁止修改

### 典型使用流程
```
1. /开始授课              # 进入授课模式
2. 进行教学答疑            # AI 基于知识库回答
3. （可选）知识考核         # 测试学习效果
4. /结束授课              # 退出授课模式
```

### 示例对话
```
/开始授课

学生：老师，刑事案件侦查期限是多久？
AI: 根据我国刑事诉讼法规定，侦查期限一般为 2 个月，
    但经上一级检察院批准可以延长。

学生：什么情况下可以延长？
AI: 对于案情复杂、期限届满不能终结的案件，
    可以延长 1 个月。

学生：这个知识点在你的知识库里吗？
AI: 是的，这是我之前系统学习过的内容。

/结束授课
# ✅ 已切换回日常模式 🟢
```

### 推荐场景
- ✅ 知识考核与测试
- ✅ 学生答疑
- ✅ 专家咨询
- ✅ 业务指导
- ✅ 培训后考核

## ⚙️ 配置参数

在 AstrBot 配置中调整：

```yaml
scriptor:
  # 学习模式配置
  learning_mode_enabled: true              # 是否启用学习模式
  learning_max_pending_items: 10           # 最大待确认知识数量
  
  # 授课模式配置
  teaching_mode_enabled: true              # 是否启用授课模式
  teaching_strict_mode: true               # 严格模式（完全禁止修改）
```

## 🎓 最佳实践

### 学习模式最佳实践

#### ✅ 推荐做法
1. **批量学习**: 一次学习 5-10 个知识点后确认
2. **及时确认**: 学习完成后及时 `/确认知识`
3. **定期检查**: 使用 `/学习状态` 检查质量
4. **修正错误**: 发现错误及时要求 AI 修改
5. **清理会话**: 学习完成后 `/结束学习`

#### ❌ 避免做法
1. ❌ 长时间不确认（待确认知识可能丢失）
2. ❌ 一次学习太多（超过最大限制）
3. ❌ 不检查质量就确认
4. ❌ 忘记退出学习模式

### 授课模式最佳实践

#### ✅ 推荐做法
1. **准备知识库**: 确保相关知识已学习
2. **严格模式**: 考核时使用严格模式
3. **记录问题**: 记录学生常问问题
4. **更新知识**: 根据问题更新知识库

#### ❌ 避免做法
1. ❌ 知识库不完整就授课
2. ❌ 在授课模式下尝试修改知识
3. ❌ 长时间不退出授课模式

## 🐛 常见问题速查

### Q1: 为什么知识没有写入数据库？
**A**: 学习模式下需要执行 `/确认知识` 才会写入。

### Q2: 待确认知识最多存多少条？
**A**: 默认 10 条，可通过 `learning_max_pending_items` 配置。

### Q3: 授课模式下真的不能修改知识吗？
**A**: 
- `teaching_strict_mode=true`: 完全禁止
- `teaching_strict_mode=false`: 会警告，但允许

### Q4: 待确认知识会过期吗？
**A**: 不会自动过期，但重启 AstrBot 后可能丢失。

### Q5: 可以在学习模式和授课模式间直接切换吗？
**A**: 可以，系统会自动处理状态转换。

### Q6: 学习模式需要管理员权限吗？
**A**: 建议管理员使用，但也可配置为所有人可用。
