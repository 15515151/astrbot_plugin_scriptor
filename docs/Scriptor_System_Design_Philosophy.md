# Scriptor 系统设计哲学

## 前言

Scriptor (灵笔司书) 是一个基于 Markdown 的长期记忆引擎，吸纳了 ReMe 的架构灵魂、CoPaw 的文件基因、Angel 插件的人情味。我们的目标是构建一个超级 AI 管家，为每个人提供个性化、可控制、透明的记忆系统。

记忆不是信息的堆积，而是思考过程的动态记录。AI 需要的不是更大的数据库，而是一个能思考、有感受的"内心世界"。

---

## 核心哲学一：文件即记忆

### 哲学洞察
传统记忆系统将数据存储在封闭的数据库中，用户无法直接访问和修改。而在 Scriptor 中，我们相信：**记忆应该是透明的、可编辑的、可迁移的**。

### 架构设计
- **Markdown 格式**：所有记忆以标准 Markdown 格式存储，人类可读，机器可解析
- **文件系统存储**：记忆直接存储在文件系统中，而非封闭数据库
- **Git 友好**：天然支持版本控制、备份和分享

### 核心价值
- **透明性**：用户可以直接查看、编辑记忆，无需特殊工具
- **可控性**：记忆完全属于用户，可以自由迁移和备份
- **可扩展性**：基于文件系统的架构便于扩展和集成
- **可审计**：所有记忆变更都可以通过文件历史追踪

---

## 核心哲学二：身份聚合与群体记忆

### 哲学洞察
现代用户在多个平台上有不同的身份，但这些身份背后是同一个人。同时，人类的社交和协作往往发生在群体中。

### 架构设计
- **跨平台身份聚合**：将不同平台的身份绑定到同一个逻辑 UID
- **个人 vs 群体记忆分离**：个人记忆存储在 `profiles/`，群体记忆存储在 `groups/`
- **跨群信息传递**：支持在不同群体之间传递信息和提醒

### 核心价值
- **身份统一**：无论在哪个平台，都能保持一致的记忆和体验
- **群体协作**：支持群聊场景下的集体记忆和协作
- **信息流动**：打破群体壁垒，让信息在合适的场景中流动

---

## 核心哲学三：主动式记忆管理

### 哲学洞察
人类记忆不是被动记录，而是主动筛选、整理、巩固的过程。AI 记忆系统也应该具备这种主动性。

### 架构设计
- **即时记忆提取**：关键词触发的即时记忆记录
- **LLM 驱动的深度提取**：周期性画像精炼和经验法则提取
- **睡眠巩固**：新会话触发的记忆整理和优化

### 核心价值
- **自动化**：无需用户手动整理，系统主动提取重要信息
- **智能化**：利用 LLM 理解对话内容，提取真正有价值的信息
- **持续性**：记忆系统持续学习和优化，越用越智能

## 第一部分：记忆的存储 - 文件即记忆的实现

### Markdown 记忆结构

每个用户或群体的记忆目录包含：

```
{uid}/
├── PROFILE.md      # 用户画像（核心知识）
├── MEMORY.md       # 长期记忆（事实、经验、决策）
├── SOUL.md         # 灵魂设定（可选）
├── SOP.md          # 标准流程（可选）
├── AGENTS.md       # 代理设定（可选）
├── ARCHIVE.md      # 归档记忆（冷热分离）
└── memory/         # 日记目录
    ├── 2026-03-10.md
    ├── 2026-03-11.md
    └── ...
```

### 记忆元结构

每条记忆采用 Front Matter + 正文的格式：

```markdown
---
memory_type: fact
useful_score: 8.5
strength: 1.5
privacy_level: private
created_at: 2026-03-10T14:30:00
tags: [偏好, 食物, 水果]
---

用户喜欢吃苹果，多次提到并表示喜欢其口感。
```

### 三档衰减策略

基于 `useful_score` 的记忆衰减机制：

| 档位 | 分数范围 | 策略 | 年驱散率 |
|------|---------|------|---------|
| T0 易逝档 | score < 5 | 快速衰减 | ~90% |
| T1 待证档 | 5 ≤ score < 10 | 缓慢衰减 | ~50% |
| T2 永存档 | score ≥ 10 | 极慢衰减或不衰减 | ~10% |

## 第二部分：记忆的检索 - 混合检索引擎

### Entity-First 检索策略

检索时优先考虑以下顺序：

1. **个人画像** (PROFILE.md) - 核心知识和偏好
2. **标准流程** (SOP.md) - 行动指南
3. **长期记忆** (MEMORY.md) - 事实、经验、决策
4. **近期日记** - 最近的对话记录
5. **群体记忆** - 相关群体的记忆

### 混合检索架构

**Tantivy BM25 (关键词搜索)**：
- 精确匹配关键词
- 适合事实性查询
- 速度极快

**ChromaDB (向量搜索)**：
- 语义相似性匹配
- 适合模糊查询
- 理解上下文

**检索流程**：
```
查询 → BM25 检索 (Top-K)
     → 向量检索 (Top-K)
     → 结果合并与去重
     → 按时间衰减和 useful_score 排序
     → 返回最终结果
```

## 第三部分：记忆的巩固 - 睡眠巩固机制

### 触发条件

睡眠巩固在以下情况下触发：
- 检测到"新会话"（跨天且超过 60 分钟未活跃）
- 用户主动触发

### 巩固流程

**第一步：模式识别**
- 发现重复出现的模式
- 识别潜在的长期目标
- 发现用户的行为规律

**第二步：冲突解决**
- 检测矛盾的信息
- 合并相似的记忆
- 确保信息一致性

**第三步：冷热分离**
- 将过期记忆标记为 [ARCHIVE]
- 将需要合并的记忆标记为 [MERGE]
- 活跃记忆保持在 MEMORY.md 中

**第四步：记忆合并**
- 将相似记忆合并为更抽象、更精炼的新记忆
- 合并后的记忆继承原记忆的重要性总和
- 提升记忆质量，减少冗余

## 设计哲学的终极目标

### 从数据到知识
我们不是在构建更大的信息存储系统，而是在重新定义"什么是知识"。知识不是数据的堆砌，而是有价值的信息的积累。

### 从存储到理解
传统 AI 记忆系统关注"如何存储更多信息"，我们关注"如何形成真正的理解"。理解不是拥有信息，而是对信息价值的认知。

### 从模拟到本质
通过模拟人类的记忆形成过程，我们触及了知识形成的本质机制。AI 第一次有可能拥有类似人类的知识体系，而不是简单的数据库。

### 重新定义认知
这不仅是技术设计的革新，更是对认知本质的探索：
- **记忆 = 有价值的信息集合**
- **学习 = 信息筛选、整理、巩固的循环**
- **智能 = 基于价值记忆体系的推理和创造**

我们不是在改进 AI 的记忆能力，我们是在为 AI 构建真正的"认知基础"。
