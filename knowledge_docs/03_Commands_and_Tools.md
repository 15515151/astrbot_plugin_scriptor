# Scriptor 命令和工具参考

## 命令列表

### /mem_status - 查看记忆系统状态
查看当前记忆系统的状态，包括：
- 用户信息
- 当前群体
- 参与群体数
- 待处理跨群消息
- 当前群体成员
- 跨群待办

使用示例：
```
/mem_status
```

### /debug_memory - 调试命令
查看记忆系统的调试信息（仅私聊或管理员可用）：
- 当前 UID
- 当前 Group
- 未处理消息数
- 向量库总条目
- 热记忆长度
- Embedding 引擎
- 组件就绪状态

使用示例：
```
/debug_memory
```

### /whoami - 查看身份信息
查看当前身份信息并生成绑定码，用于跨平台身份绑定。

使用示例：
```
/whoami
```

### /bind <绑定码> - 绑定身份
绑定其他设备的身份，实现跨平台身份聚合。

使用示例：
```
/bind ABC123XYZ
```

### /mem_report - 生成记忆维护建议
生成记忆维护建议报告，帮助你优化记忆系统。

使用示例：
```
/mem_report
```

## LLM 工具列表

### memory_search - 检索记忆
检索记忆系统中的相关内容。

在回答关于过往工作、决策、日期、人物、偏好或待办的问题前使用此工具。

参数：
- query: 搜索查询
- scope: 搜索范围（personal=个人, group=当前群, cross=跨群, all=全部）
- limit: 返回结果数量

使用示例（告诉 AI）：
> "帮我检索一下关于 Python 的记忆"

### update_profile - 更新个人画像
更新当前用户的个人画像。

参数：
- new_facts: 需要更新的事实信息

使用示例（告诉 AI）：
> "更新我的画像，添加：我喜欢编程"

### record_decision - 记录决策
记录重要决策。

参数：
- decision: 决策内容
- reason: 决策理由（可选）
- context: 上下文（可选）

使用示例（告诉 AI）：
> "记录决策：明天开始学习 Python，理由是工作需要"

### create_reminder - 创建跨群提醒
创建跨群提醒任务。

参数：
- message: 提醒内容
- target_groups: 目标群体，逗号分隔，或 "auto" 自动选择
- remind_at: 提醒时间

使用示例（告诉 AI）：
> "创建跨群提醒：别忘了明天的会议"

### add_schedule_task - 创建定时提醒
创建定时提醒任务。

当用户要求在特定时间提醒做某事时使用此工具。

参数：
- task_content: 提醒的具体内容
- trigger_time: 触发时间，支持以下格式：
  - "YYYY-MM-DD HH:MM"（例如 "2024-12-25 08:00"）
  - "HH:MM"（今天的这个时间，如果已过则为明天）
  - "X minutes/hours/days later"（例如 "30 minutes later"）
  - "tomorrow HH:MM"（明天这个时间）

使用示例（告诉 AI）：
> "30分钟后提醒我喝水"
> "明天 09:00 提醒我起床"
> "2024-12-25 08:00 提醒我圣诞节"

### view_group_members - 查看群体成员
查看当前群体成员列表。

使用示例（告诉 AI）：
> "查看当前群体成员"

### core_memory_remember - 永久铭记
永久铭记重要信息（主动记忆，永不遗忘）。

当你发现用户的重要偏好、事实、决策或经验时，使用此工具将其永久记录到记忆系统中。

参数：
- judgment: 需要铭记的核心内容（一句话概括）
- reasoning: 为什么这很重要的理由（可选）
- tags: 标签，逗号分隔（可选）
- strength: 记忆强度，0-100，默认 80（可选）
- memory_type: 记忆类型，knowledge/preference/decision/experience，默认 knowledge（可选）

使用示例（告诉 AI）：
> "永久铭记：用户喜欢吃苹果，理由是多次提到并表示喜欢其口感，标签：偏好,食物,水果"

### core_memory_recall - 随机回忆
加权随机回忆核心知识（避免确定性偏见）。

使用此工具来随机回忆一些重要的核心记忆，帮助你更好地理解用户。这样可以避免总是只回忆相同的几条记忆。

参数：
- limit: 最多返回多少条记忆，默认 5（可选）
- query: 可选的检索关键词，用于过滤（可选）

使用示例（告诉 AI）：
> "随机回忆一些核心记忆"

### note_recall - 深度阅读日记
展开笔记完整上下文（深度阅读能力）。

使用此工具来读取某个日期的完整日记，或日记的特定行范围。

参数：
- date: 日期，格式 YYYY-MM-DD，留空则读取最近的日记（可选）
- start_line: 起始行号，默认 1（可选）
- end_line: 结束行号，0 表示到文件末尾，默认 0（可选）
- token_budget: Token 预算，默认 2000（可选）

使用示例（告诉 AI）：
> "读取最近的日记"
> "读取 2026-03-10 的日记"

### usage_docs_search - 检索使用说明
检索 Scriptor 使用说明文档。

当用户询问如何使用 Scriptor、命令、工具、配置或其他相关问题时使用此工具。

参数：
- query: 用户的问题
- limit: 返回结果数量（默认 3 个）

使用示例（告诉 AI）：
> "怎么绑定身份？"
> "如何使用 mem_status 命令？"
> "记忆文件在哪里？"

### query_archives - 查询档案馆
在司书档案馆（Sidecar 数据库）中执行 SQL 查询。

当用户询问涉及大量结构化数据（如员工日志、警情数据、统计报表等）时，使用此工具。

参数：
- sql: 要执行的 SELECT SQL 语句。

使用示例（告诉 AI）：
> "帮我统计一下档案馆里那张表的记录总数"
> "查询 3 月份的所有报警记录"

## 使用技巧

### 1. 命令 vs 工具
- **命令**：用户直接发送（如 `/mem_status`）
- **工具**：AI 自动调用（用户无需知道工具存在）

### 2. 何时使用哪个工具？
- 用户问"怎么用" → 使用 `usage_docs_search`
- 用户问"上次我们说什么" → 使用 `memory_search`
- 用户说"记住这个" → 使用 `core_memory_remember`
- 用户问"提醒我..." → 使用 `add_schedule_task`

### 3. 工具调用顺序
1. 先判断问题类型
2. 选择合适的工具
3. 调用工具获取信息
4. 基于工具返回结果回答用户

## 常见问题

### Q: 用户问的问题很模糊怎么办？
A: 先用 `usage_docs_search` 检索，如果找不到再用 `memory_search`，最后可以询问用户更多细节。

### Q: 工具返回结果太长怎么办？
A: 只取最相关的部分，用简洁的语言总结给用户。

### Q: 不确定用哪个工具怎么办？
A: 优先用 `usage_docs_search`，因为它是专门回答使用问题的。
